# Plan: DataLoader 강화 - 국내외 주식/지수 데이터 수집

## 1. 개요

| 항목 | 내용 |
|------|------|
| Feature | data-loader-enhancement |
| 작성일 | 2026-03-06 |
| 상태 | Plan |
| 우선순위 | High |
| 대상 파일 | src/data_loader.py |

### 1.1 목적
기존 DataLoader를 확장하여 국내외 주요 지수(KOSPI, S&P 500 등)와 개별 종목 데이터를 통합 수집하고, 메모리 캐싱과 CSV 파일 캐싱을 모두 지원하는 고성능 데이터 수집 모듈로 강화한다.

### 1.2 현재 상태 (AS-IS)
- 기본적인 fetch/save/load/get 기능만 존재
- 지수 데이터 미지원
- 메모리 캐싱 없음
- 캐시 유효성 검증 없음

### 1.3 목표 상태 (TO-BE)
- 주요 지수 프리셋 지원 (KOSPI, KOSDAQ, S&P500, NASDAQ, DOW)
- 개별 종목 코드 기반 데이터 수집 (005930 등)
- 2단계 캐싱: 메모리 캐시 (dict) + CSV 파일 캐시
- 캐시 만료 관리 (일 단위)
- 복수 종목 일괄 다운로드

## 2. 기능 요구사항

### FR-01: 지수 데이터 프리셋
- KOSPI, KOSDAQ, S&P500, NASDAQ, DOW 등 주요 지수 코드 매핑
- `fetch_index('KOSPI', start, end)` 형태로 간편 호출

### FR-02: 개별 종목 데이터
- 종목 코드 직접 입력 (예: '005930', 'AAPL')
- 기간별 OHLCV 데이터 수집

### FR-03: 메모리 캐싱
- dict 기반 인메모리 캐시
- 동일 종목 반복 조회 시 API 재호출 방지
- 캐시 초기화 메서드 제공

### FR-04: CSV 파일 캐싱
- data/ 폴더에 종목별 CSV 저장
- 캐시 만료일 지정 (기본 1일)
- 만료된 캐시 자동 갱신

### FR-05: 복수 종목 일괄 다운로드
- 종목 코드 리스트 입력으로 여러 종목 한번에 수집
- dict[ticker, DataFrame] 형태로 반환

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 성능 | 캐시 히트 시 즉시 반환 (< 1ms) |
| 안정성 | API 오류 시 기존 캐시 데이터 반환 |
| 호환성 | 기존 DataLoader 인터페이스 하위 호환 유지 |
